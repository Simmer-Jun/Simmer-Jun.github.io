##最近一道题目是说深度克隆对象##
但是自己貌似对于类型的判断还是有些不清楚–
索性用最笨的方法全都alert()了一遍，以下见Code

结论

  • typeof 用于判断基本类型 number/string/boolean/null(object)/undefined/object;返回的是字符串”number””string”等
  • typeof(fn),fn是一个函数，返回结果是function 所以可以使用typeof 与 instanceof 判断函数
  • 使用new操作生成的对象则需要使用 obj instanceof str (str=Function/Array/Date/Error/RegExp)判断,返回布尔值。（注意instanceof不能用来判断使用new操作符构造的 Number String 对象）对于ECMASript6新增的symbol?还不熟悉,暂不涉猎,留着下回–

更新结论（补充）

  • 使用object.prototype.toString.call()可以准确的判断引用类型（解决跨ifram的问题）
  • 在较老的chrome浏览器上，正则表达式(/a/)使用’typeof’判断的结果是’function’。因此使用toString的方式判断更准确。
