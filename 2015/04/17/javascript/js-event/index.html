事件的分类

事件是JavaScript跳动的心脏，通过JavaScript我们可以监听特点的事件，在事件发生的时候执行相应的js程序以作出反应。
在今天的事件中，由于历史的原因，事件分为DOM标准的事件模型（支持事件捕获与冒泡型）和IE特有的事件模型（冒泡型事件）。但是由于W3C标准的推进，目前的DOM标准已经将IE的冒泡型事件吸收到其中。
*注意，目前支持DOM标准的浏览器（Firefox，chrome,Safari,Opera)等都支持DOM标准的事件模型（addEventListener(type,handler,false/true)其中false指的是事件在冒泡阶段被调用，true值得是事件在捕获阶段被调用），而IE仍然使用自己专有的冒泡型事件模型attach(“on”+type,handler)。

停止事件冒泡和阻止事件的默认行为

引用作者原话：
“停止事件冒泡“和”阻止浏览器的默认行为“，这两个概念非常重要，它们对复杂的应用程序处理非常有用。
1.停止事件冒泡
停止事件冒泡是指，停止冒泡型事件的进一步传递（取消事件传递,不只是停止IE和DOM标准共有的冒泡型事件，我们还可以停止支持DOM标准浏览器的捕捉型事件，用topPropagation()方法）。例如上图中的冒泡型事件传递中，在body处理停止事件传递后，位于上层的document的事件监听器就不再收到通知，不再被处理。
2.阻止事件的默认行为
停止事件的默认行为是指，通常浏览器在事件传递并处理完后会执行与该事件关联的默认动作（如果存在这样的动作）。例如，如果表单中input type 属性是 “submit”，点击后在事件传播完浏览器就自动提交表单。又例如，input 元素的 keydown 事件发生并处理后，浏览器默认会将用户键入的字符自动追加到 input 元素的值中。
