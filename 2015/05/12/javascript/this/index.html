我理解的this

JavaScript 中函数的调用有以下几种方式：作为对象方法调用，作为函数调用，作为构造函数调用，和使用 apply 或 call 调用。

在JavaScript中没有其他面对对象编程语言中的类的概念，而是以其特有的原型链方式来实现’类’的继承。而说到继承优不得不谈到JavaScript中的this关键字。

我个人理解的就是this指代的就是函数运行时函数所在的环境。这个和函数的闭包类似但有有些不同（恰恰相反），我们知道一个函数运行在它所定义的环境，而不是运行他的环境（详情可以看这里）。

  • 在JS中, 是有预编译的过程的, JS在执行每一段JS代码之前, 都会首先处理var关键字和function定义式(函数定义式和函数表达式)，也就是说你可以在定义函数之前允许函数(但是一般不推荐这样做==)。

  • JavaScript 中的函数既可以被当作普通函数执行，也可以作为对象的方法执行，这是导致 this 含义如此丰富的主要原因。一个函数被执行时，会创建一个执行环境（ExecutionContext），函数的所有的行为均发生在此执行环境中，构建该执行环境时，JavaScript 首先会创建 arguments变量，其中包含调用函数时传入的参数。接下来创建作用域链。然后初始化变量，首先初始化函数的形参表，值为 arguments变量中对应的值，如果 arguments变量中没有对应值，则该形参初始化为 undefined。如果该函数中含有内部函数，则初始化这些内部函数。如果没有，继续初始化该函数内定义的局部变量，需要注意的是此时这些变量初始化为 undefined，其赋值操作在执行环境（ExecutionContext）创建成功后，函数执行时才会执行，这点对于我们理解 JavaScript 中的变量作用域非常重要。
    this变量赋值，会根据函数调用方式的不同，赋给 this全局对象，当前对象等。至此函数的执行环境（ExecutionContext）创建成功，函数开始逐行执行，所需变量均从之前构建好的执行环境（ExecutionContext）中读取。

好了，看完上面的话，我假定你已经了解了函数执行与变量预编译（其实真正深入会知道，在函数执行前的预编译过程就会对函数内定义的变量进行预定义，然后在执行的时候去逐条执行语句）。很显然我们先定义了一个Gobal对象name，在函数sayHello里先console.log(this.name)，这里的this指代的就是全局对象（注意在严格模式时this指代的是undefined）。

做一个小总结

  • this指代的往往是当前运行环境所在的作用域。
  • 一般的全局函数中this指向global对象
  • 对象方法中的this指向当前对象（注：当对象方法赋值给变量时，要看这个变量函数运行时所在的环境）
  • 构造函数中的this会指向实例化的对象。
  • 使用call和apply可以改变当前函数的this指向，这里面大有文章，包括js原生实现bind方法都是用到call和apply方法
  • 在setInterval和setTimeout中，this会指向global对象。
  • global对象是ECMAScript规定的全局对象，但在浏览器中的这个全局对象一般指代window
