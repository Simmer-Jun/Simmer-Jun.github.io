css2.1

width的取值与overflow


今天（5/27/2015）有位小伙伴问了一个关于div溢出父元素的问题，仔细想一下，发现自己也不是很懂，于是就花时间把各种width overflow都试了一下。demo就暂时不发了，写下总结吧，感兴趣的童鞋可以回头试一下。

首先回顾一下基础知识：

知识点1 overflow

  • 可能的值有5种，分别是visible/hidden/scroll/auto/inherit.
  • 元素的默认值为visible，也就是超出的内容不会被修剪，会呈现在元素框之外。
  • hidden，内容会被修剪，并且其余内容是不可见的。
  • scroll，内容会被修剪，但是浏览器会显示滚动条以便查看其余的内容。
  • auto，如果内容被修剪，则浏览器会显示滚动条以便查看其余的内容。
  • inherit，规定应该从父元素继承 overflow 属性的值。(这里看出overflow默认是不被继承的)
  • 不得不要提一下：overflow值不为visible的话会触发元素生成BFC，BFC对于理解页面布局十分重要（可以用来闭合浮动，看那些年我们一起清除过的浮动）。关于BFC你可以看这里了解

知识点2 关于width的取值

你需要知道以下几点：

  • 当width="a%"时，元素width取值是其直接父元素的width*a%。
  • 默认情况下width=auto。

知识点3 关于盒模型

  • 只有width margin可以设定为auto。
  • 设定margin: 0 auto会使得元素在父盒子中水平居中（垂直方向设置auto会重置为0，但是对于定位元素来说，其处理有所不同，感兴趣的童鞋可以去试一下哈~）。
  • 不得不谈到margin值可以为负值的 这个特点，经常被用来布局，例如在容器里面水平垂直居中等。
  • 设置背景时候，背景覆盖的范围包括padding在内，并不包括margin。

width的取值与overflow关系？

  • 当父元素的宽度是定宽，即其父元素设定width=apx时：
    • 子元素width=auto（默认值，当没有显式的设置宽度时），子元素将会尽可能调整自己的宽度已使得充满父容器（margin-left+border-left+padding-left+width+padding-right+border-right+margin-right等于父元素宽度apx，貌似有个box-sizing的属性，可以使得父元素width=子元素width）。
    • 子元素width=b%子元素宽度将严格按照父元素宽度来设定宽度（apx*b%）。
  • 当父元素的宽度是auto时，这个比较复杂，还要分析父元素的其他值得设定。不过这里有关于csswidth与margin组合在取值的详细分析。

css hack总结

前段时间偶然看到一些前端面试题，其中一道考察的是css hack知识，感觉自己对于css hack理解的还是不够透彻，所以写下来作为对自己知识的补充，如果能帮到你，那我将十分高兴。

什么是css hack

这里引用百度百科：

CSS hack由于不同厂商的浏览器，比如Internet Explorer,Safari,Mozilla Firefox,Chrome等，或者是同一厂商的浏览器的不同版本，如IE6和IE7，对CSS的解析认识不完全一样，因此会导致生成的页面效果不一样，得不到我们所需要的页面效果。 这个时候我们就需要针对不同的浏览器去写不同的CSS，让它能够同时兼容不同的浏览器，能在不同的浏览器中也能得到我们想要的页面效果。

这里要先声明几点：

  • css hack是前端工程师必备的技能，但并不意味着我们要尽肯能多的去使用它，相反我们要极力避免使用。
  • 一个优秀的前端工程师应该推崇前端标准化，而非一味的去使用hack（当然为了网站在不同浏览器看起来一样我们不得不这样做）。

css hack 分类

  • 属性前缀法(即类内部Hack)：例如 IE6能识别下划线”“和星号” “，IE7能识别星号” “，但不能识别下划线”“，IE6~IE10都认识”\9”，但firefox前三个都不能认识。
  • 选择器前缀法(即选择器Hack)：即在css选择器前加上特有的前缀，使得不同的浏览器匹配不同的规则。

  • IE条件注释法(即HTML条件注释Hack)：针对不同的IE(注：IE10+已经不再支持条件注释)使用不同的css样式表。
